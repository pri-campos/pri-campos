# SIM-004 — Partial Failure (Saga Interruption)

## Objective

Simulate partial failure in distributed transaction flows to validate whether compensation mechanisms restore system consistency.

## How to simulate

- Interrupt execution mid-flow in multi-step processes
- Fail specific services during transaction execution
- Simulate rollback scenarios

## What it stresses

- saga orchestration
- compensation logic
- distributed consistency

## Expected invariants

- System must recover to a consistent state
- Partial execution must not leave orphan states
- Compensation must be correctly applied

## Failure signals

- orphan transactions
- inconsistent states across services
- failed or missing compensation

## Related risks

- Architecture / lack of coordination in distributed flow
- Architecture / ambiguity of responsibility in compensation
- Architecture / fragmented ownership of transaction state
- Architecture / inconsistent handling of partial execution
- Reliability / failure recovery issues
- Data consistency / incomplete transactions
- Business-financial / inconsistent effects

## Notes

Common in distributed systems where atomic transactions are not possible.
