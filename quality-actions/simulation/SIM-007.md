# SIM-007 — Asynchronous Contract Break

## Objective

Simulate contract violations in asynchronous communication to verify system resilience and isolation.

## How to simulate

- Send messages with invalid schema
- Introduce incompatible event versions
- Break expected message format

## What it stresses

- schema validation
- contract enforcement
- system isolation

## Expected invariants

- Invalid messages must not break the system
- Consumers must handle unexpected inputs safely
- System must remain stable

## Failure signals

- crashes in consumers
- cascading failures
- unhandled exceptions

## Related risks

- Architecture / lack of contract ownership definition
- Architecture / weak boundary enforcement between components
- Architecture / insufficient validation at system boundaries
- Architecture / propagation of invalid state across components
- Integration / contract mismatch
- Reliability / system fragility
- Security / malformed input handling

## Notes

Important for systems with evolving contracts and multiple producers/consumers.
