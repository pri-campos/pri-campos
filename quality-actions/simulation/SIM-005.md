# SIM-005 — Dependency Latency Injection

## Objective

Introduce artificial latency in dependencies to verify system behavior under slow or degraded external services.

## How to simulate

- Add delays to external service calls
- Increase response times artificially
- Simulate timeouts

## What it stresses

- timeout handling
- retry mechanisms
- resource management

## Expected invariants

- System must fail safely under latency
- No resource leakage or blocking behavior
- Flow must degrade gracefully

## Failure signals

- cascading failures
- blocked transactions
- excessive retries

## Related risks

- Architecture / tight coupling and dependency sensitivity
- Architecture / lack of isolation between components
- Architecture / propagation of failures across boundaries
- Architecture / excessive reliance on synchronous chains
- Integration / dependency fragility
- Reliability / timeout handling failure
- Customer experience / degraded performance

## Notes

Critical for systems dependent on external providers like payment gateways.
