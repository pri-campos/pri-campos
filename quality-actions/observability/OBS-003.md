# OBS-003 — Latency Outlier Detection

## Objective

Identify abnormal latency patterns across components in a transaction flow.

## What to observe

- latency between services (hop latency)
- spikes in response time
- slow propagation across events

## What it reveals

- bottlenecks
- resource contention
- degraded dependencies

## Expected invariants

- latency should remain within acceptable thresholds
- propagation time should be consistent

## Failure signals

- latency spikes
- inconsistent response times
- delayed processing

## Related risks

- Architecture / tight coupling impacting latency propagation
- Architecture / lack of isolation between critical and non-critical paths
- Architecture / excessive synchronous dependencies in critical flow
- Architecture / inefficient coordination across distributed components
- Integration / dependency degradation
- Reliability / performance instability
- Customer experience / slow transactions

## Notes

Latency issues often precede system failures.
