# OBS-001 — Dangling Transactions Detection

## Objective

Detect transactions that start but do not reach a terminal state within an expected timeframe.

## What to observe

- transactions without terminal state after SLA
- long-running intermediate states
- missing completion events

## What it reveals

- partial failures
- broken flows
- missing transitions

## Expected invariants

- every transaction must reach a terminal state
- transitions must progress within expected time

## Failure signals

- orphan transactions
- stalled flows
- missing final state

## Related risks

- Architecture / lack of control over transaction lifecycle completion
- Architecture / absence of clear terminal state definition
- Architecture / broken coordination in state transitions
- Architecture / missing ownership of flow completion
- Reliability / incomplete execution
- Data consistency / incomplete state
- Customer experience / stuck transactions

## Notes

Critical for detecting silent failures that do not trigger explicit errors.
