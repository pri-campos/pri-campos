# OBS-005 — Event Sequencing Anomalies

## Objective

Detect events arriving or being processed out of logical order.

## What to observe

- out-of-order events
- unexpected state transitions
- sequence mismatches

## What it reveals

- race conditions
- ordering issues
- asynchronous inconsistencies

## Expected invariants

- events must follow logical sequence
- state transitions must respect ordering

## Failure signals

- invalid transitions
- unexpected state jumps
- inconsistent sequencing

## Related risks

- Architecture / lack of control over event ordering and state transitions
- Architecture / ambiguity in authority over state progression
- Architecture / inconsistent coordination across asynchronous components
- Architecture / violation of deterministic state evolution
- Integration / ordering issues
- Data consistency / invalid transitions
- Reliability / async coordination

## Notes

Common in event-driven systems.
