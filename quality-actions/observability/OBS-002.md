# OBS-002 — Semantic Divergence Detection

## Objective

Detect cases where logically identical transactions produce different outcomes.

## What to observe

- same idempotency key with different payloads
- inconsistent results across retries
- conflicting transaction states

## What it reveals

- violation of source of truth
- inconsistent business logic
- idempotency failures

## Expected invariants

- same logical input must produce same output
- transaction identity must be consistent

## Failure signals

- divergent results
- conflicting state updates
- inconsistent financial effects

## Related risks

- Architecture / lack of single source of truth
- Data consistency / divergence
- Business-financial / inconsistent outcomes

## Notes

Important for detecting hidden inconsistencies in distributed systems.
